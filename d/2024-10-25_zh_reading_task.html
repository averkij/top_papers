Breaking the Memory Barrier: Near Infinite Batch Size Scaling for Contrastive Loss

1. 这篇文章讨论了对比损失在表示学习中的应用。

2. 大批量数据可以提高性能，但受限于GPU内存消耗。

3. 作者提出了一种基于块的计算策略，避免全部实例化相似度矩阵。

4. 此外，他们引入了多级块策略，利用分布式系统的层次结构。

5. 实验结果显示，该方法可以扩展到前所未有的批量大小，并保持精度。

6. 例如，使用8或32个A800 80GB，可以训练CLIP-ViT-L/14模型，批量大小为4M或12M。

7. 与最先进的内存高效解决方案相比，它在保持相当速度的同时，实现了两个数量级的内存减少。

8. 代码将公开发布。

1. This article discusses the application of contrastive loss in representation learning.

2. Large-scale data can improve performance but is limited by GPU memory consumption.

3. The authors propose a block-based computation strategy to avoid fully instantiating the similarity matrix.

4. Additionally, they introduce a multi-level block strategy that leverages the hierarchical structure of distributed systems.

5. Experimental results show that this method can scale to unprecedented batch sizes while maintaining accuracy.

6. For example, using 8 or 32 A800 80GB GPUs, the CLIP-ViT-L/14 model can be trained with batch sizes of 4M or 12M.

7. Compared to state-of-the-art memory-efficient solutions, it achieves a two-order-of-magnitude reduction in memory usage while maintaining comparable speed.

8. The code will be made publicly available.
